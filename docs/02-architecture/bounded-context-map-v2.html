Key Decisions

  • Ideas are seeds authored by individuals — they can be discussed and forked, but not formally amended.
  • Forking an idea produces a new Idea with a SourceIdea reference for credit/lineage tracking.
  • A Project is a user group's active working version — it may optionally reference a source Idea.
  • Amendments govern changes to a project's description and goals, governed by the UserGroup's Charter (specifically its ApprovalRules) — not the parent Society's SocialContract.
  • Society's SocialContract and UserGroup's Charter are deliberately modelled as different things — the former governs rights and membership, the latter governs operational decision-making.
  • Requirements are updated independently by users — separate from the amendment process.
  • A UserGroup governs exactly one Commons system; a Commons can have multiple UserGroups, including competing ones with different approaches.

Open Questions

  • Can a Project be created directly by a user group without a source Idea, or must it always trace back to one?
  • Reputation / trust graph BC (Neo4j, planned) not yet shown.
  • UserGroup membership is currently flat — Marsh's model implies concentric tiers (inner epistemic community with amplified voice, outer affected parties). Membership tier modelling not yet designed.
